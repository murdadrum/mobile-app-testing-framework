## Project Structure
- docs/ - documentation and test strategy
- src/ - implementation or sample code
- tests/ - automated test suites
- scripts/ - helper scripts
- .github/workflows/ - CI placeholders

## Getting Started
1. Review the test strategy in docs/TEST_STRATEGY.md.
2. Add project-specific setup steps and dependencies.
3. Extend src/ and tests/ with real implementations.
